Inkapsling ​

Inledning ​

Inkapsling är ett grundläggande begrepp inom objektorienterad programmering som handlar om att dölja en objekts interna data och endast tillåta åtkomst till den via väl definierade metoder. Genom att kapsla in data med privata attribut kan vi skydda den från oavsiktlig eller obehörig manipulation, vilket ökar säkerheten i koden.

1. Vad är inkapsling? ​

Inkapsling innebär att dölja en objekts interna tillstånd (attribut) och endast ge åtkomst via metoder (funktioner inom en klass). På så sätt kan man skydda viktiga data från att förändras direkt utanför klassen och säkerställa att endast avsedd funktionalitet kan påverka datan.

Inkapsling genomförs i OOP med hjälp av två huvudbegrepp:

  1. Attribut och metoder som privata eller skyddade.
  2. Getter- och setter-metoder (get och set funktioner) som styr åtkomsten till dessa privata eller skyddade attribut

2. Privata, skyddade och publik attribut ​

I Python använder vi en konvention för att markera attribut som privata eller skyddade:

  • Ett skyddat attribut skrivs med ett understreck _ framför namnet (t.ex. _name), vilket indikerar att det inte bör ändras utanför klassen, men det är inte strikt förbjudet att göra så.
  • Ett privat attribut skrivs med två understreck __ framför namnet (t.ex. __securityNr personnummer). Detta hindrar åtkomst till attributet från utanför klassen med hjälp av namnändring (name mangling).
  • Ett public attribut kan nås och användas både inifrån och utanför klassen (t.ex. age)

Exempel på privata, skyddade och publika attribut

python
class Person:
    def __init__(self, social_security_number, name, age):
        self.__securityNr = social_security_number  # Privat attribut
        self._name = name                           # Skyddat attribut
        self.age = age                              # Publik attribut

    def show_info(self):
        print(f"{self._name} har personnummer {self.__securityNr} och är {self.age} år!")  # Publik metod


# ANVÄNDNING

# Skapa ett Person-objekt
person = Person("20060912-2314", "Kalle Anka", 18)
person.show_info()

# Vi kan direkt komma åt det publika attributet-
print(person.age)

# Vi kan direkt komma åt det skyddade attributet (även om det inte är rekommenderat)
print(person._name)

# Försök att direkt komma åt det privata attributet orsakar ett fel
#print(person.__securityNr)  # Ger AttributeError

# Men vi kan fortfarande nå det privata attributet genom namnändring (name mangling)
print(person._Person__securityNr)  # Direkt åtkomst till privat attribut

3. Getter- och setter-metoder ​

För att kontrollera åtkomsten till privata attribut, skapar vi metoder som tillåter oss att läsa eller skriva till dessa attribut på ett säkert sätt. Dessa metoder kallas getter (get_attribute) och setter (set_attribute).

Exempel på getters och setters:

python
class Car:
    def __init__(self, model, price):
        self.__model = model        # Privat attribut
        self.__price = price        # Privat attribut

    # Getter för modell
    def get_model(self):  
        return self.__model

    # Setter för modell
    def set_model(self, new_model):
        self.__model = new_model

    # Getter för pris
    def get_price(self):
        return self.__price
    
    # Setter för pris (med extra logik för att validera priset)
    def set_price(self, new_price):
        if new_price > 0:
            self.__price = new_price
        else:
            print("Priset måste vara positivt.")


# ANVÄNDNING

# Skapa ett Car-objekt
car = Car("Volvo", 300000)

Getter och setter-metoder låter oss kontrollera hur ett attribut uppdateras och hämtas, vilket ger bättre kontroll över datan och skyddar den från felaktiga värden.

4. Exempel ​

Vi ska skapa en klass som representerar ett Bankkonto. Vi använder privata variabler för att skydda känslig data (som saldot), getter och setter-metoder för att kontrollera åtkomsten till dessa variabler, och en privat metod som hanterar interna operationer.

Kodexempel:

python
class Bankkonto:
   def __init__(self, kontoinnehavare, saldo):
       self.__kontoinnehavare = kontoinnehavare  # Privat variabel (kontoinnehavare)
       self.__saldo = saldo                      # Privat variabel (saldo)

   # Getter-metod för kontoinnehavare
   def get_kontoinnehavare(self):
       return self.__kontoinnehavare

   # Setter-metod för kontoinnehavare (om vi tillåter att ändra kontoinnehavare)
   def set_kontoinnehavare(self, ny_innehavare):
       if ny_innehavare:
           self.__kontoinnehavare = ny_innehavare
       else:
           print("Ogiltigt kontonamn!")

   # Getter-metod för saldo
   def get_saldo(self):
       return self.__saldo

   # Metod för att sätta in pengar (med logik)
   def insättning(self, belopp):
       if belopp > 0:
           self.__saldo += belopp
           print(f"{belopp} kr insatta.")
       else:
           print("Beloppet måste vara större än 0.")

   # Metod för att ta ut pengar (med kontroll)
   def uttag(self, belopp):
       if belopp > 0 and belopp <= self.__saldo:
           self.__saldo -= belopp
           print(f"{belopp} kr uttagna.")
           # Anropa en privat metod för att kontrollera om lågt saldo har uppnåtts
           self.__kontrollera_lågt_saldo()
       else:
           print("Ogiltigt uttagsbelopp eller otillräckligt saldo.")

   # Privat metod för att kontrollera om saldot är lågt
   def __kontrollera_lågt_saldo(self):
       if self.__saldo < 100:
           print(F"Varning: Lågt saldo! Du har mindre än 100 kr kvar på kontot.")
       else:
           print(F"Saldo OK.")

# ANVÄNDNING

print(f"-----------------------------")
print(f"\tBANKKONTO ")
print(f"-----------------------------")

# Visa kontoinnehavare och saldo

Förklaring av Kodexemplet: ​

  1. Privata variabler:

    • __kontoinnehavare och __saldo är två privata variabler som representerar kontoinnehavarens namn och kontots saldo. Dessa variabler är skyddade från direkt åtkomst utanför klassen.

  2. Getter- och setter-metoder:

    • get_kontoinnehavare() används för att få kontoinnehavarens namn.
    • set_kontoinnehavare() tillåter att kontoinnehavarens namn uppdateras. Den innehåller också validering för att säkerställa att det nya namnet inte är tomt.
    • get_saldo() returnerar kontots saldo.
    • Inga setters används för att direkt ändra saldot, istället används metoder som insättning() och uttag() för kontrollerad ändring.

  3. Publika metoder för kontohantering:

    • insättning() tillåter att pengar sätts in på kontot och uppdaterar saldot.
    • uttag() tillåter uttag av pengar, men endast om beloppet är giltigt och inte överstiger saldot. Efter uttaget kontrolleras om saldot är lågt genom att anropa en privat metod.

  4. Privat metod:

    • __kontrollera_lågt_saldo() är en privat metod som kontrollerar om saldot har sjunkit under 100 kr. Denna metod kallas internt från uttag() när ett uttag görs. Genom att hålla denna metod privat kan vi skydda den från att användas utanför klassen.

5. Fördelar med inkapsling ​

  1. Skydd av data: Inkapsling skyddar viktiga attribut från att oavsiktligt ändras från externa delar av koden.
  2. Kontrollerad åtkomst: Du kan styra hur ett objekts interna tillstånd ändras genom att skapa metoder med valideringslogik.
  3. Ökad säkerhet: Genom att dölja interna detaljer kan du undvika fel och potentiella säkerhetsrisker som kan uppstå om data ändras på fel sätt.

Sammanfattning ​

  • Innkapsling är en central del av objektorienterad programmering som hjälper till att skydda data och säkerställa att den endast kan ändras på ett kontrollerat sätt.
  • I Python används attribut med understreck _ och __ för att markera dem som skyddade eller privata.
  • Getter- och setter-metoder används för att ge kontrollerad åtkomst till dessa attribut, vilket gör din kod säkrare och enklare att underhålla.
