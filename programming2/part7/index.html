Arv ​

Arv är en viktig del av Python som gör det möjligt för oss att skapa nya klasser baserade på redan existerande klasser.

Vad är Arv? ​

Definition ​

Arv i objektorienterad programmering är en mekanism där en ny klass (kallas subklass eller barnklass) kan ärva egenskaper (attribut och metoder) från en annan klass (kallas superklass eller föräldraklass). Detta innebär att en subklass får alla egenskaper från sin superklass, men den kan också lägga till egna attribut och metoder eller modifiera de som redan finns.

Varför är Arv viktigt? ​

  1. Återanvändning av kod: Arv gör det möjligt att återanvända kod från en klass i en annan, vilket sparar tid och ansträngning.
  2. Underhåll: Genom att använda arv kan vi göra ändringar i superklassen som automatiskt gäller för alla dess subklasser, vilket underlättar underhåll.
  3. Struktur och Organisation: Arv hjälper till att organisera och strukturera koden genom att skapa en hierarki av klasser.

Grunderna i Arv ​

Tänk dig att vi ska skapa ett system för att hantera olika typer av fordon, som bilar och cyklar. Vi börjar med att skapa en superklass Vehicle (Fordon), och sedan skapar vi subklasser Car (Bil) och Bike (Cykel) som ärver från Vehicle.

Steg 1: Skapa Superklassen Vehicle (Fordon) ​

Vi börjar med att definiera superklassen Vehicle (Fordon) som kommer att innehålla grundläggande egenskaper och metoder för alla fordon:

Exempel

  • Skapa en Python-fil som heter Vehicles.py,
  • kopiera klassen Vehicle nedan och klista in i Python-filen Vehicles.py
python
class Vehicle:
    def __init__(self, brand, model, year):
        self.brand = brand
        self.model = model
        self.year = year
    
    def start(self):
        return "Fordonet startar."

    def stop(self):
        return "Fordonet stannar."

    def description(self):
        return f"{self.year} {self.brand} {self.model}"

Steg 2: Skapa Subklassen Car (Bil) ​

Nu skapar vi en subklass Car (Bil) som ärver från Vehicle (Fordon). Subklassen Car kommer att ha några ytterligare attribut och metoder som är specifika för bilar.

Exempel

  • kopiera klassen Car nedan och klista in i Python-filen Vehicles.py
python
class Car(Vehicle):
    def __init__(self, brand, model, year, number_of_doors):
        super().__init__(brand, model, year)
        self.number_of_doors = number_of_doors

    def open_luggageCompartment(self):  #bagageutrymme
        return "Bagageutrymmet öppnas."
    
    def description(self):
        return f"{self.year} {self.brand} {self.model} med {self.number_of_doors} dörrar"

Steg 3: Skapa Subklassen Bike (Cykel) ​

Sedan skapar vi en subklass Bike (Cykel) som också ärver från Vehicle, men den har sina egna egenskaper:

Exempel

  • kopiera klassen Bike nedan och klista in i Python-filen Vehicles.py
python
class Bike(Vehicle):
    def __init__(self, brand, model, year, type_of_bike):
        super().__init__(brand, model, year)
        self.type_of_bike = type_of_bike

    def ring_bell(self):
        return "Pling pling!"
    
    def description(self):
        return f"{self.year} {self.brand} {self.model} - en {self.type_of_bike} cykel"

Steg 4: Använda Klasserna ​

Nu när vi har våra klasser, låt oss skapa några objekt och se hur de fungerar:

Exempel

  • Skapa en Python-fil som heter main.py,
  • kopiera koden nedan och klista in i Python-filen main.py,
  • kör igång/exekvera programmet!
python
from Vehicles import *

my_car = Car("Volvo", "XC40", 2023, 4)
my_bike = Bike("Crescent", "Modell X", 2022, "Mountainbike")

print(my_car.description())   # Output: 2023 Volvo XC40 med 4 dörrar
print(my_car.start())         # Output: Fordonet startar.
print(my_car.open_luggageCompartment())  # Output: Bagageutrymmet öppnas.

print(my_bike.description())  # Output: 2022 Crescent Modell X - en Mountainbike cykel
print(my_bike.start())        # Output: Fordonet startar.
print(my_bike.ring_bell())    # Output: Pling pling!

super() funktion ​

Funktionen super() i Python används för att anropa metoder från en superklass (föräldraklass) i en subklass (barnklass). Den låter dig återanvända och utöka funktioner som redan finns i superklassen, istället för att skriva om dem.

Varför Använda super()? ​

  • Återanvändning: Med super() kan du använda och bygga vidare på kod som redan finns i superklassen.
  • Exempel: Om en bilklass ärver från en fordonsklass, kan super() användas för att få tillgång till fordonsklassens funktioner, som att starta motorn, samtidigt som du lägger till bilspecifika saker, som att öppna bagagedörren.

Utöka Funktionalitet med super() ​

Obs!

Om vi vill åsidosätta en metod från superklassen men ändå använda en del av dess funktionalitet, kan vi använda super().

python
class Car(Vehicle):

    def description(self):
        base_description = super().description()
        return f"{base_description} med {self.number_of_doors} dörrar"

Sammanfattning ​

Arv är en kraftfull mekanism inom objektorienterad programmering som gör det möjligt att återanvända kod, strukturera program och skapa hierarkier av klasser. Genom att förstå arv kan vi skapa mer komplexa och funktionella program i Python. Vi har gått igenom grunderna i arv, hur man skapar super- och subklasser, och hur man hanterar flerfaldig arv.

För att bemästra arv är det viktigt att öva och experimentera med flera övningar/uppgifter.
