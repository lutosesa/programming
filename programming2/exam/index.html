1. Objekt Orienterad Programmering ​

Definition:

Objektorienterad programmering (OOP) är ett sätt att skriva program som organiserar koden runt objekt istället för bara funktioner och logik. Ett objekt är en sak i programmet som kan representera verkliga saker, som en bil, en bok eller en användare. Varje objekt har egenskaper (attribut) och kan utföra handlingar (metoder).

I OOP använder vi klasser för att skapa objekt som är instanser av klasser.


2. Klasser och objekt ​

Definition:

En klass är en mall för att skapa objekt. Ett objekt är en instans av en klass som representerar en specifik instans med attribut (data) och *metoder (funktioner).

Exempel:

python
class Person:
    def __init__(self, name, age):
        self.name = name    # instansattribut
        self.age = age      # instansattribut    

    def display(self):
        print(f"Name: {self.__name}, Age: {self.__age}")

# Exempel på användning
# Skapa ett objekt
person = Person("Allan", 18)
print(person.name, person.age)
person.display()        # anropar objektets metod

3. Inkapsling (Encapsulation) ​

Definition:

Inkapsling är principen att begränsa tillgången till vissa delar av ett objekt. Detta görs vanligtvis med privata attribut och metoder.
Ja, inkapsling innebär att dölja en objekt attribut och endast ge åtkomst via metoder.

getter och setter ​

För att komma åt privata attribut, skapar vi metoder som tillåter oss att läsa eller skriva till dessa attribut på ett säkert sätt. Dessa metoder kallas gettet (get_attribute) och setter (set_attribute).

Exempel:

python
class Person:
    def __init__(self, name, age):
        self.__name = name   # privat instansattribut
        self.__age = age     # privat instansattribut

    # Getter för name
    def get_name(self):
        return self.__name

    # Setter för name
    def set_name(self, name):
        if len(name) > 0:   # Kontroll för att namnet inte är tomt
            self.__name = name
        else:
            print("Namn kan inte vara tomt.")

    # Getter för age
    def get_age(self):
        return self.__age

    # Setter för age
    def set_age(self, age):
        if age > 0:  # Kontroll för att åldern måste vara ett positivt tal
            self.__age = age
        else:
            print("Ålder måste vara ett positivt tal.")

    # Metod för att visa personens information
    def display(self):
        print(f"Name: {self.__name}, Age: {self.__age}")

# Exempel på användning
# Skapa ett Person-objekt
person1 = Person("Emil", 18)

# Visa personens initiala information
person1.display()

# Ändra namn och ålder med hjälp av setter-metoder
person1.set_name("Axel")
person1.set_age(19)
person1.display()       # Visa uppdaterad information

# Försök att sätta ett ogiltigt namn och ålder
person1.set_name("")  # Ogiltigt namn (tom sträng)
person1.set_age(-5)   # Ogiltig ålder (negativt tal)

  1. Privata attribut:

  2. Getter-metoder:
    • Getter-metoderna get_name() och get_age() returnerar de privata attributen __name och __age.
  3. Setter-metoder:

  4. Visa information:
    • Metoden display() används för att skriva ut personens namn och ålder.

@property ​

@property är en inbyggd dekorator (en funktion) i Python som används för att hantera åtkomsten till ett klassattribut på ett rent och enkelt sätt.

Exempel:

python
class Person:
    def __init__(self, name, age):
        self.__name = name   # privat instansattribut
        self.__age = age     # privat instansattribut

    # Getter för name med @property
    @property
    def name(self):
        return self.__name

    # Setter för name med @name.setter
    @name.setter
    def name(self, value):
        if len(value) > 0:   # Kontroll för att namnet inte ska vara tomt
            self.__name = value
        else:
            print("Namn kan inte vara tomt.")

    # Getter för age med @property
    @property
    def age(self):
        return self.__age

    # Setter för age med @age.setter
    @age.setter
    def age(self, value):
        if value > 0:  # Kontroll för att åldern måste vara ett positivt tal
            self.__age = value
        else:
            print("Ålder måste vara ett positivt tal.")

    # Metod för att visa personens information
    def display(self):
        print(f"Name: {self.__name}, Age: {self.__age}")


# Exempel på användning
person = Person("Anton", 18)

# Visa personens initiala information
person.display()

# Ändra namn och ålder med hjälp av @property-metoder
person.name = "Josep"      # Använder setter-metoden automatiskt
person.age = 19            # Använder setter-metoden automatiskt
# Visa uppdaterad information
person.display()

# Försök att sätta ett ogiltigt namn och ålder
person.name = ""               # Namn kan inte vara tomt
person.age = -5                # Ålder måste vara positivt

4. Arv (Inheritance) ​

Definition:

Arv gör att en klass kan ärva attribut och metoder från en annan klass, vilket möjliggör återanvändning och strukturering av kod.

Exempel:

python
class Person:
    def __init__(self, name, age):
        self.name = name    # instansattribut, instansvariabel
        self.age = age      # instansattribut, instansvariabel   

    def introduce(self):
        print(f"Jag heter {self.name} och är {self.age} år gammal.")

class Student(Person):
    def __init__(self, name, age, person_number):
        super().__init__(name, age)  # Anropar basklassens __init__ metod
        self.person_number = person_number

    def display(self):
        print(f"Jag heter {self.name} och är {self.age} år gammal. Mitt personnummer är {self.person_number}.")


# Exempel på ANVÄNDNING:
student = Student("Allan", 18, "20061018-2450")
print(student.name, student.age, student.person_number)

# ändrar värde på namn, age och person_number
student.name = "Alfred"
student.age = 20
student.person_number = "2004-01-10-2578"

# anropar objektets metod 
student.display()

5. Klasshierarkier ​

Definition:

En klasshierarki beskriver strukturen där klasser är ordnade i nivåer baserat på arv. I objektorienterad programmering innebär det att en underklass (subklass) ärver egenskaper och metoder från en överklass (superklass). Klasshierarkier kan ha flera nivåer, där varje underklass bygger vidare på sin överklass och kan lägga till eller modifiera funktionalitet.

Exempel

python
# Nivå 1: Superklass Person
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    # Metod för att visa personens information
    def display(self):
        print(f"Name: {self.name}, Age: {self.age}")


# Nivå 2: Subklass Employee som ärver från Person
class Employee(Person):
    def __init__(self, name, age, employee_id):
        super().__init__(name, age)  # Anropar Person-konstruktorn
        self.employee_id = employee_id

    # Metod för att visa anställdes information
    def display(self):
        super().display()  # Anropar display från Person
        print(f"Employee ID: {self.employee_id}")


# Nivå 3: Subklass Manager som ärver från Employee
class Manager(Employee):
    def __init__(self, name, age, employee_id, department):
        super().__init__(name, age, employee_id)  # Anropar Employee-konstruktorn
        self.department = department

    # Metod för att visa chefens information
    def display(self):
        super().display()  # Anropar display från Employee
        print(f"Department: {self.department}")


# Exempel på ANVÄNDNING:
person   = Person("Luke", 25)
employee = Employee("Hugo", 30, "E123")
manager  = Manager("Eva", 35, "M456", "HR")

person.display()    # Output: Name: Luke, Age: 25
print()             # Tom rad för läsbarhet
employee.display()  # Output: Name: Hugo, Age: 30, Employee ID: E123
print()             # Tom rad för läsbarhet
manager.display()   # Output: Name: Eva, Age: 35, Employee ID: M456, Department: HR

6. Polymorfism ​

Definition:

Polymorfism innebär att olika klasser kan implementera metoder med samma namn på olika sätt. Detta gör koden mer flexibel och återanvändbar

Exempel

python
# Superklass Person
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    # En metod som subklasserna kommer att anpassa
    def display_role(self):
        pass

# Subklass Student som ärver från Person
class Student(Person):
    def __init__(self, name, age, student_id):
        super().__init__(name, age)
        self.student_id = student_id

    # Anpassar display_role för Student
    def display_role(self):
        print(f"Student: {self.name}, ID: {self.student_id}")

# Subklass Teacher som ärver från Person
class Teacher(Person):
    def __init__(self, name, age, subject):
        super().__init__(name, age)
        self.subject = subject

    # Anpassar display_role för Teacher
    def display_role(self):
        print(f"Lärare: {self.name}, Ämne: {self.subject}")


# Exempel på ANVÄNDNING:
def introduce_person(person):
    person.display_role()  # Anropar rätt version beroende på objektets typ

# Skapa objekt av typerna Student och Teacher
student = Student("Eva Berg", 18, "S12345")
teacher = Teacher("Ola Hjärpe", 45, "Matematik")

# Polymorfism: Både Student och Teacher anropar sin egen version av display_role
introduce_person(student)   # Output: Student: Eva Nerg, ID: S12345
introduce_person(teacher)   # Output: Lärare: Ola Hjärpe, Ämne: Matematik

7. Abstrakta klasser och metoder ​

Definition:

En abstrakt klass är en klass som inte kan instansieras (man kan inte skapa objekt direkt från den). Den fungerar som en mall för subklasser som ärver den. En abstrakt metod är en metod som definieras i en abstrakt klass men inte har någon implementation. Subklasser måste implementera de abstrakta metoderna.

I Python används modulen abc (Abstract Base Class) för att skapa abstrakta klasser.

Exempel

python
from abc import ABC, abstractmethod

# Abstrakt superklass Person
class Person(ABC):
    def __init__(self, name, age):
        self.name = name
        self.age = age

    # Abstrakt metod (måste implementeras i subklasser)
    @abstractmethod
    def display_role(self):
        pass


# Subklass Student som ärver från Person
class Student(Person):
    def __init__(self, name, age, student_id):
        super().__init__(name, age)
        self.student_id = student_id

    # Implementering av den abstrakta metoden
    def display_role(self):
        print(f"Student: {self.name}, ID: {self.student_id}")


# Subklass Teacher som ärver från Person
class Teacher(Person):
    def __init__(self, name, age, subject):
        super().__init__(name, age)
        self.subject = subject

    # Implementering av den abstrakta metoden
    def display_role(self):
        print(f"Lärare: {self.name}, Ämne: {self.subject}")


# Exempel på ANVÄNDNING:
student = Student("Lucky ", 18, "S12345")
teacher = Teacher("Omer Malkic", 45, "Matematik")

student.display_role()   # Output: Student: Alice, ID: S12345
teacher.display_role()   # Output: Lärare: Omer Malkic, Ämne: Matematik

8. Generiska klasser (Generics) ​

Definition:

Generiska klasser och metoder gör det möjligt att skriva kod som fungerar med olika datatyper, utan att vara specifik för en viss typ. Generiska metoder kan ta emot eller returnera vilken typ som helst, och generiska klasser kan lagra eller manipulera objekt av olika typer. Detta gör koden mer flexibel och återanvändbar.

Python stödjer generiska klasser genom typing-modulen.

Exempel

python
from typing import TypeVar, Generic

# Definiera en generisk typ T
T = TypeVar('T')

# Generisk metod för att skriva ut vilken typ som helst
def print_item(item: T):
    print(f"Item: {item}, Type: {type(item).__name__}")

# Generisk klass Storage (Lagring) som kan lagra vilken typ som helst
class Storage(Generic[T]):
    def __init__(self):
        self.items = []

    # Lägg till ett objekt av valfri typ
    def add_item(self, item: T):
        self.items.append(item)

    # Hämta alla objekt
    def get_items(self):
        return self.items

# En enkel Person-klass
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def __str__(self):
        return f"Name: {self.name}, Age: {self.age}"


# Exempel på ANVÄNDNING:
# Använda generisk metod med primitiva datatyper
print_item(100)         # Output: Item: 100, Type: int
print_item("Hello")     # Output: Item: Hello, Type: str

# Använda generisk metod med en egen datatyp (Person)
person = Person("Emil", 20)
print_item(person)      # Output: Item: Name: Emil, Age: 20, Type: Person

# Använda generisk klass med primitiva datatyper
int_storage = Storage[int]()  # Skapa Storage-objekt för int
int_storage.add_item(10)
int_storage.add_item(20)
print(f"Lagrade heltal: {int_storage.get_items()}")  # Output: Stored ints: [10, 20]

# Använda generisk klass med egen datatyp (Person)
person_storage = Storage[Person]()  # Skapa Storage-objekt för Person
person_storage.add_item(Person("Allan", 25))
person_storage.add_item(Person("Luke", 20))

for person in person_storage.get_items():
    print(person)  # Output: Name: Sven, Age: 25 | Name: Kalle, Age: 20

Sammanfattning ​

Objektorienterad programmering i Python innehåller flera kraftfulla begrepp:

  1. Objektorienterad programmering (OOP): är en programmeringsmetod där man organiserar kod i objekt som innehåller både data (attribut) och funktioner (metoder). Dessa objekt är instanser av klasser.
  2. Klasser och objekt: Grunderna i OOP, där klasser skapar objekt.
  3. Inkapsling: Skyddar data genom att begränsa åtkomst till vissa attribut.
  4. Arv: Möjliggör återanvändning av kod genom att ärva attribut och metoder.
  5. klasshierarkier: är strukturer där klasser ordnas i nivåer, där subklasser ärver egenskaper och metoder från överordnade klasser (superklasser). Detta möjliggör återanvändning av kod och specialisering.
  6. Polymorfism: Tillåter att metoder med samma namn har olika implementationer beroende på objektet.
  7. Abstrakta klasser: Definierar gemensamt beteende men kan inte instansieras, och subklasser måste implementera abstrakta metoder.
  8. Generiska metoder och klasser: Gör det möjligt att skapa typoberoende klasser och metoder.

Dessa begrepp gör det lättare att strukturera och återanvända kod, vilket leder till bättre underhållbara och skalbara program.
