Projekt 5 : Buzzer ​

Inledning ​

Vi ska koppla in en analog piezosummer ( buzzer) som kan spela olika toner och styrs därför INTE med på- och avslag av spänning. Istället används den inbyggda funktionen tone() som skickar ut en önskad ton (frekvens) under önskad tid. Du ska skapa en egen funktion som anropar den inbyggda funktionen tone().

Ja, tone() kan enbart generera fyrkantsvågor och saknar möjlighet att justera ljudstyrkan (volym). Det går därför inte att få någon hifi-upplevelse, men det är inga problem att spela enkla melodier.
För att spela melodier behövs två saker:

  • en lista över vilka toner som motsvarar vilka frekvenser
  • en melodi (en lista med toner).

Du ska använda dig av en egen funktion som du kommmer att skapa för att ange/skapa ljud för piezosummern.

Syfte med projektet ​

Du kommer att lära dig om:

  • hur man skapar en egen funktion i Arduino. En egen funktion som INTE returnerar något.
  • hur man anropar den egna funktionen.
  • hur de huvudfunktionerna och funktioner som tone() och noTone() används.
  • Hur INPUT_PULLUP används och ansluts till Arduino kretsen och vad det innebär denna koppling.
  • hur man kan styra dessa elektroniska komponenter genom att koda fram en algoritm på utvecklingsmiljön Arduino IDE.

Vad är en Piezosummrar (buzzers)? ​

En piezosummer (buzzer) i Arduino är en liten elektronisk komponent som omvandlar elektriska signaler till ljud. Den kan generera enkla ljudsignaler, toner eller pip genom att vibrera en piezoelektrisk kristall inuti.
Piezosummrar används ofta för att ge ljudfeedback i olika projekt, till exempel för att signalera larm, spela musik eller skapa ljud för knapptryck.
I Arduino styrs en piezosummer enkelt med funktionen tone() för att skapa olika frekvenser (toner) och noTone() för att stänga av ljudet.

Elektroniska delar som behövs ​

  • (1) Arduino Uno
  • (1) USB A-to-B kabel
  • (1) Kopplingsplatta
  • (1) piezosummer
  • sladdar

Vad är en funktion i Arduino? ​

En funktion i Arduino är en del av koden som utför en specifik uppgift. Funktioner gör koden mer organiserad och lättare att förstå, eftersom du kan återanvända samma kod flera gånger. Det finns två huvudtyper av funktioner i Arduino:

  1. Inbyggda funktioner: Dessa är redan definierade i Arduino-språket, som t.ex.:
  • setup(): Körs en gång när programmet startar och används för att initiera inställningar.
  • loop(): Körs kontinuerligt och innehåller programmets huvudlogik.
  1. Användardefinierade funktioner: Du kan också skapa dina egna funktioner för specifika uppgifter. De definieras med syntaxen:
cpp
// Skapar funtionen 
void getBuzzer() {
   // Kod som körs när funktionen anropas
}

// Anropar funktionen
getBuzzer();

Övning 1 ​

Vi ska koppla in en analog piezosummer (buzzer) på den vita kopplingsplattan till kretskortet Arduino Uno.

Kopplings ritning ​

Programkod ​

cpp
/******************************************
  Projekt 5 - Övning 1 - Buzzer
  En buzzer/summer skapar enkla ljud. 

  http://www.arduino.cc/en/Tutorial/Tone
********************************************/
int buzzer = A1;
int timer = 300;

void setup() {
} 

void loop() {
   getBuzzer();
}

void getBuzzer(){
  tone(buzzer, 490); //Skapar en ton(ljud)
  delay(timer);
  noTone(buzzer);    //stänger av ljudet.
  delay(timer);
  
  timer = timer + 600;
}

Övning 2 ​

Nu ska du ansluta en tryckknappbrytare till kretsen för att slå på och av piezosummern.

Klicka här för förklaringen om:   Vad är och gör INPUT_PULLUP i Arduino?

INPUT_PULLUP i Arduino gör det enkelt att koppla en knapp eller annan ingång utan att behöva extra komponenter, som motstånd. Det är ett smart läge för ingångspinnar som gör att Arduino automatiskt sätter en pin till HIGH (på) när ingen annan signal är kopplad.

Vad gör INPUT_PULLUP?

När du kopplar en knapp till en Arduino utan INPUT_PULLUP, kan ingången vara osäker när knappen inte trycks. Den kan växla mellan HIGH och LOW, vilket kallas ett "flytande" tillstånd.

När du ställer in en pin som INPUT_PULLUP, aktiverar Arduino ett inbyggt motstånd som gör att ingångspinnen alltid är HIGH när knappen INTE är intryckt. När du trycker på knappen och kopplar pinnen till jord (GND), ändras den till LOW.

Exempel:

  • Utan att trycka på knappen: Pinnen är HIGH (1).
  • När knappen trycks: Pinnen blir LOW (0).

Varför är det bra?
Du slipper använda extra motstånd i kretsen eftersom Arduino har det inbyggt!

Kopplings ritning ​

Programkod ​

cpp
/******************************************
  Projekt 5 - Övning 2 - Buzzer
  
  När Tryckknappen trycks ner går igång
  piezosummer/buzzer och spelar en melodi.  
********************************************/
// tonerna i melodin: 
#define NOTE_C4  262
#define NOTE_G3  196
#define NOTE_A3  220
#define NOTE_B3  247
int melody[] = {262, 196, 196, 220, 196, 0, 247, 262};
 
// tonens längd: 4 = kvarton, 8 = åttonde ton, etc.:
int noteDurations[] = {4, 8, 8, 4, 4, 4, 4, 4};

// Skapar 'const' variabler för buzzern pinnr och knappens pinne
const int BUZZER = A1; 
const int PIN_BUTTON = 4;

void setup() {
    pinMode(PIN_BUTTON, INPUT_PULLUP);  
    Serial.begin(9600);
}

void loop() {
    int pushValue = digitalRead(PIN_BUTTON);
    
    Serial.print("pushValue = ");
    Serial.println(pushValue);
    
    if(pushValue == LOW){
        myMelodi();
    }
    else {
        noTone(buzzer);
    }
}

void myMelodi(){ 
  // loopar igenom över meloditoner: 
  for (int thisNote = 0; thisNote < 8; thisNote++) {

    // ta en sekund för att beräkna tonens längd
    // dividerat med tontypen.
    // t.ex. kvartton = 1000/4, åttonde ton = 1000/8, etc. 
    int noteDuration = 1000 / noteDurations[thisNote];
    tone(buzzer, melody[thisNote], noteDuration);

    // för att skilja tonerna, ställ in en minimitid mellan dem.
    // tonens längd + 30% verkar fungera bra: 
    int pauseBetweenNotes = noteDuration * 1.30;
    delay(pauseBetweenNotes);

    // sluta spela tonen:
    noTone(buzzer);
  }
}
