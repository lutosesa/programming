Laboration: Blink lysdiod ​

Inledning ​

Du ska ansluta en röd Lysdiod och en resistor på den vita kopplingsplattan till kretskortet Arduino Uno. Lysdioden ska tändas en sekund och sedan släckas en sekund. Sedan kommer du att koppla in en gul lysdiod och senare en grön lysdiod för att simulera ett trafikljus.

Syfte med labbet ​

Du kommer att lära dig om:

  • hur variabler deklareras och tilldelas
  • hur används de huvudfunktionerna och andra inbyggda funktioner som ingår i programmet.
  • hur man kopplar in en lysdiod och en resistor,
  • hur man kan styra dessa elektroniska komponenter genom att koda fram en algoritm på utvecklingsmiljön i Tinkercad.

Vad är en variabel och en datatyp? ​

En variabel är som en låda där du kan lagra data, som ett tal eller text som du vill använda senare i ditt program. Du kan tänka på det som en plats där du kan spara något och sedan använda det igen när du behöver.

En datatyp beskriver vilken typ av data variabeln kan lagra. Det kan vara:

  • int för heltal, som 10 eller -5.
  • float för decimaltal, som 3.14.
  • char för enskilda tecken, som 'A'.
  • String för text, som "Hej".

Ja, en variabel lagrar data, och datatypen bestämmer vilken sorts data det är.

Vad är en lysdiod? ​

  • Lysdioden är en elektrisk komponent som släpper igenom ström åt ena hållet.

  • Den har den speciella egenskapen att den börjar lysa när ström går igenom den.

  • För att indikera hur den ska monteras är dess ena ben längre än det andra.

  • Det längre benet är anoden, det vill säga benet som ska kopplas till pluspolen och det korta benet katoden kopplas till minuspolen.

Vad är en resistor (motstånd)? ​

Resistorns huvuduppgifter är att sänka spänningen och begränsa strömmen i en krets. Du kan se dem som att de "gör sladden smalare" så att de släpper igenom mindre ström. Det har ingen betydelse åt vilket håll du vänder resistorn (motståndet).

Elektroniska delar som behövs ​

  • (1) Arduino Uno
  • (1) USB A-to-B kabel
  • (1) Kopplingsplatta
  • (1) Röd lysdiod (LED)
  • (1) Gul lysdiod (LED)

Övning 1 ​

  • Ansluta en röd Lysdiod och en resistor på den vita kopplingsplattan till kretskortet Arduino Uno. slut
  • Anslut lysdiodens långa pinne till utgångsstift 13
  • Anslut lysdiodens korta pinne till resistorn som i sin tur kopplas till minus
  • Lysdioden ska tändas en sekund och sedan släckas en sekund.

Kopplings ritning ​

Programkod ​

För att kopiera koden nedan, klicka på den vänstra ikonen nedan (cpp).

cpp
/****************************************************
* Labb - övning 1 - Blink Lysdiod
* 
* Programmet får en lysdiod (på pin 13) att blinka 
* med en sekund mellan varje blinkning.
*****************************************************/

void setup()
{
  pinMode(13, OUTPUT);
}

void loop()
{
  digitalWrite(13, HIGH);   //Lysdioden tänds
  delay(1000);              //väntar en sekund 
  digitalWrite(13, LOW);    //Lysdioden släcks
  delay(1000);              //väntar en sekund
}

Här är en enkel förklaring av vad programmet gör:

  1. setup():
    • Funktionen pinMode(13, OUTPUT) sätter pin 13 som UTSIGNAL, så Arduino kan styra lysdioden.
  2. loop():
    • digitalWrite(13, HIGH) tänder lysdioden på pin 13.
    • delay(1000) gör att programmet väntar 1 sekund.
    • digitalWrite(13, LOW) släcker lysdioden.
    • delay(1000) väntar ytterligare 1 sekund innan loopen börjar om.

Sammanfattning: Lysdioden blinkar på och av med en sekunds mellanrum.

Övning 2 ​

I programkoden ska du deklarera och tilldela:

  • en global variabel för pin 13 och
  • en vanlig variabel för tid,

Det vill säga, du ska skapa en global variabel för pinnen som går till lysdioden och ange dess värde (13), samt en variabel för sekunderna. Variablernas namn ska beskriva vad de gör.

För att kopiera koden nedan, klicka på den vänstra ikonen nedan (cpp).

cpp
/*******************************************
* Labb - övning 2 - Blink Lysdiod
********************************************/

// skapa en variabel för den anslutna pinnen
const int PIN_LED_RED = 13;

// skapa en variaber för timer
int timer = 1000;

void setup()
{
  pinMode(PIN_LED_RED, OUTPUT);
}

void loop()
{  
  digitalWrite(PIN_LED_RED, HIGH);  
  delay(timer);                     
  digitalWrite(PIN_LED_RED, LOW);   
  delay(timer);                    
}

Här är en enkel förklaring av vad programmet ovan gör:

  1. Konstant variabel (rad 9):
    • är en variabel vars värde inte kan ändras efter att den har deklarerats. Detta görs genom att använda nyckelordet const. Namnet på en konstant brukar oftast anges med versaler (stora bokstäver) för att skilja den från vanliga variabler, men detta är en konvention, inte ett krav.
  2. Vanlig variabel (rad 12):
    • En vanlig variabel deklareras utan const och dess värde kan ändras under programmets gång. Den används för att hålla data som kan variera under exekveringen. Namn på vanliga variabler brukar skrivas med små bokstäver eller kamelnotation t.ex. ledPin.

Övning 5 ​

Nu har du tre lysdioder på kopplingsplattan kopplade till kretskortet Arduino Uno, en röd, gul och grön lysdiod.
Vi ska simulera ett trafikljus på kopplingsplattan och efterlikna trafiksignaler:


  1. Röd lysdiod ska vara tänd 6 sekunder, och både den gula och gröna dioden ska vara släckta.

  2. Sedan ska den röda och gula lysdioden vara tända två sekunder och den gröna lysdioden ska var släckt.

  3. Sedan ska den gröna lysdioden vara tänd 6 sekunder och både den röda och gula ska vara släckta.

  4. Sedan ska den gula vara tänd två sekunder och både den gröna och röda ska vara släckta.

Programkod: ​

cpp
/******************************************
* Labb - övning 5 - Blink Lysdiod
*
* Programmet simulerar ett trafikljus.
*******************************************/

// skapa variabler för de anslutna pinnarna 
const int PIN_LED_RED    = 13;
const int PIN_LED_YELLOW = 12;
const int PIN_LED_GREEN  = 13;

// skapa variabler för två timer
int timer_6s = 6000;
int timer_2s = 2000;

void setup()
{
  pinMode(PIN_LED_RED,    OUTPUT);
  pinMode(PIN_LED_YELLOW, OUTPUT);  
  pinMode(PIN_LED_GREEN,  OUTPUT); 
}

void loop()
{ 
    digitalWrite(PIN_LED_RED,    HIGH); 
    digitalWrite(PIN_LED_YELLOW, LOW);                             
    digitalWrite(PIN_LED_GREEN,  LOW);      
    delay(timer_6s);                         
    
    // Här nedan skapas de andra tre steg
    digitalWrite(PIN_LED_RED,    HIGH);   //Röd  lysdioden tänds
    digitalWrite(PIN_LED_YELLOW, HIGH);   //Gul  lysdioden tänds.
    digitalWrite(PIN_LED_GREEN,  LOW);    //Grön lysdioden släcks
    delay(timer_2s);						          //väntar 2 sekunder.
    
    digitalWrite(PIN_LED_RED,    LOW);    //Röd  lysdioden släcks
    digitalWrite(PIN_LED_YELLOW, LOW);    //Gul  lysdioden släcks.
    digitalWrite(PIN_LED_GREEN,  HIGH);   //Grön lysdioden tändss
    delay(timer_6s);						          //väntar 6 sekunder.
    
    digitalWrite(PIN_LED_RED,    LOW);   	//Röd  lysdioden släcks
    digitalWrite(PIN_LED_YELLOW, HIGH);   //Gul  lysdioden tänds.
    digitalWrite(PIN_LED_GREEN,  LOW);    //Grön lysdioden släcks
    delay(timer_2s);                      //väntar 2 sekunder.
}

Övning 6 ​

Vi kommer att gå igenom denna övning senare under kursens gång!

Man upprepar liknande kod flera gånger vilken det är inte så effektivt och flexibel. Man kan koda färre rader, mer lättläst och enklare att underhålla med en array som är listor, for-loop och skapa en egen funktion för varje steg i traffikljuset.

cpp
// Skapa en array för pinnarna för röd, gul och grön lysdiod
const int LED_PINS[] = {13, 12, 11}; 

// Skapa en variabel för antal lysdioder
const int NUMBER_OF_LEDS = 3;       

// Skapa variabler för timer
int timer_6s = 6000; 
int timer_2s = 2000;

void setup() 
{
  // Använder en for-loop för att sätta pinnarna för lysdioder som UTSIGNALER.
  for (int i = 0; i < NUM_LEDS; i++) {
    pinMode(LED_PINS[i], OUTPUT);
  }
}

// Skapar en egen funktion för att hantera varje steg i trafikljuset
void SetTrafficLight(int redState, int yellowState, int greenState, int delayTime) 
{
  digitalWrite(LED_PINS[0], redState);    // Ställ in röd lysdiod
  digitalWrite(LED_PINS[1], yellowState); // Ställ in gul lysdiod
  digitalWrite(LED_PINS[2], greenState);  // Ställ in grön lysdiod
  delay(delayTime);                       // Vänta angiven tid
}

void loop() 
{
  // Anropa funktionen för varje steg i trafikljussekvensen
  SetTrafficLight(HIGH, LOW,  LOW,  timer_6s);  // Röd på, Gul av, Grön av
  SetTrafficLight(HIGH, HIGH, LOW,  timer_2s);  // Röd på, Gul på, Grön av
  SetTrafficLight(LOW,  LOW,  HIGH, timer_6s);  // Röd av, Gul av, Grön på
  SetTrafficLight(LOW,  HIGH, LOW,  timer_6s);  // Röd av, Gul på, Grön av
}

Förklaring av koden ​

  1. Funktion SetTrafficLight:

    • Denna funktion tar fyra parametrar:

      • redState: Tillståndet (HIGH eller LOW) för den röda lysdioden.
      • yellowState: Tillståndet för den gula lysdioden.
      • greenState: Tillståndet för den gröna lysdioden.
      • delayTime: Hur länge detta steg ska pågå.

    • Inuti funktionen ställer vi in LED tillstånd med digitalWrite och väntar sedan med delay.

  2. Anrop av funktionen i loop: -I loop() anropas funktionen SetTrafficLight fyra gånger, en gång för varje steg i trafikljussekvensen. Varje anrop anger de specifika tillstånden för lysdioderna och hur länge varje tillstånd ska pågå.

Fördelar med denna lösning ​

  • Modularisering: Koden är nu modulär, vilket innebär att du enkelt kan återanvända och ändra sekvenserna utan att behöva duplicera kod.
  • Enkelhet: Det blir tydligt vilka steg som utförs i loop, och det är lätt att ändra varje steg genom att justera funktionsanropen.
  • Tydlighet: Funktionen SetTrafficLight gör att huvudloopen (loop) blir mycket mer lättläst och överskådlig.

Denna metod gör att du enkelt kan skala upp eller ned, till exempel genom att lägga till fler lysdioder eller fler steg, utan att behöva skriva om stora delar av koden.
