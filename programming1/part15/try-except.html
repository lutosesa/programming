Python try except block ​

Välkommen till denna genomgång om att använda try-except block i Python! Vi kommer att lära oss hur man hanterar fel (även kallade undantag) i dina Python-program på ett elegant sätt. Målet är att ge dig en solid förståelse av try-except block, så att du kan skriva mer robust och stabil kod.

1. Vad är undantag (exception)? ​

Undantag (exception) är fel som uppstår under körning av ett program. Till skillnad från syntaxfel, som upptäcks av tolken eller kompilatorn innan programmet körs, uppstår undantag under körning och kan leda till att programmet kraschar om de inte hanteras.

Exempel på vanliga undantag:

  • ZeroDivisionError: Försöker dela ett tal med noll.
  • IndexError: Försöker komma åt ett index som inte finns i en lista.
  • KeyError: Försöker komma åt en nyckel som inte finns i en ordbok.

2. Varför behövs try-except block? ​

För att undvika att program kraschar när undantag inträffar och istället kunna hantera dessa situationer på ett kontrollerat sätt. Genom att använda try-except block kan vi:

  • Fånga och hantera specifika fel.
  • Ge användaren tydliga felmeddelanden.
  • Fortsätta köra programmet även efter att ett fel inträffat.

3. Grundläggande struktur för try-except block ​

Strukturen för ett try-except block ser ut så här:

python
try:
    # kod som kan orsaka ett undantag
except Undantagstyp:
    # kod som körs om undantaget inträffar

4. Hantera flera undantag ​

Vi kan hantera flera olika typer av undantag i samma try-except block genom att använda flera except-satser.

Exempel på att hantera flera undantag:

python
try:
    number = int(input("Ange ett nummer > "))
    resultat = 10 / number
    print("Resultat:", resultat)
except ValueError:
    print("Du måste ange ett giltigt nummer!")
except ZeroDivisionError:
    print("Du kan inte dela ett tal med noll!")

5. else och finally med try-except ​

Vi kan också använda else och finally med try-except block för att specificera kod som ska köras om inget undantag inträffar eller oavsett om ett undantag inträffar.

Exempel med else ​

Ex:

python
try:
    number = int(input("Ange ett nummer: "))
    resultat = 10 / number
except ZeroDivisionError:
    print("Du kan inte dela ett tal med noll!")
except ValueError:
    print("Du måste ange ett giltigt nummer!")
else:
    print("Resultat:", resultat)

6. Skapa egna undantag (överkurs) ​

Ibland kan det vara användbart att skapa egna undantag för att hantera specifika fel i ditt program. Detta görs genom att skapa en ny klass som ärver från Exception.

Exempel:

Låt oss säga att vi vill skapa ett program som kontrollerar om en användares ålder är inom ett visst intervall. Om åldern är utanför detta intervall, vill vi skicka vårt eget undantag.

python
import os
import time

class MyOwnException(Exception):
    """En egen undantagsklass för specifika fel"""
    def __init__(self, message):
        self.message = message
        super().__init__(self.message)

# Funktionen kontrollerar åldern
def check_age(age):
    if age < 0 or age > 120:
        raise MyOwnException("Åldern måste vara mellan 0 och 120!")
    else:
        print(f"Din ålder är {age} år.")

# funktionen tömmer terminalen
def clear_console():
    os.system('cls' if os.name == 'nt' else 'clear')

# Kör programmet "för alltid"   
while True:
    try:
        age = int(input("\nAnge din ålder (-1 för att avsluta) > "))
        
        # Avslutar programmet
        if age == -1:
           break
         
        check_age(age)
        
    except MyOwnException as e:
        print(f"Ett fel inträffade: {e}")
    except ValueError:
        print("Du måste ange ett giltigt nummer!")
        
# ANSI escape-sekvenser för färgad text
RED, BLUE, GREEN, CYAN = "\033[31m", "\033[34m", "\033[32m", "\033[36m"
colors = [RED, BLUE, GREEN, CYAN, RED,]

clear_console()
for i in range(5):
    print(f"\n{colors[i]}Välkommen tillbaka !")
    time.sleep(2)     # Vänta en sekund
    clear_console()       # Töm konsolen

Förklaring av exemplet

  1. Definiera klassen: Vi definierar klassen MyOwnException som ärver från Exception och har en _init_ metod för att ta emot ett felmeddelande.
  2. Funktion som använder undantaget: Funktionen check_age kontrollerar om åldern är inom det tillåtna intervallet. Om åldern är utanför intervallet skickar den ett MyOwnException med ett specifikt meddelande.
  3. Try-except block: Vi använder ett try-except block för att fånga och hantera undantaget. Om ett MyOwnException uppstår, skrivs ett anpassat felmeddelande ut. Om användaren anger något som inte är ett giltigt nummer (vilket skulle orsaka ett ValueError), hanteras det också.

Avlutning ​

Att använda try-except block är en viktig färdighet för att hantera fel i dina Python-program. Genom att fånga och hantera undantag kan du göra dina program mer robusta och användarvänliga.

Kom ihåg att alltid testa din kod noggrant och tänka på möjliga fel som kan uppstå, så att du kan hantera dem på ett lämpligt sätt.
